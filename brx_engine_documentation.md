# Documentação Técnica e Guia de Uso da BRX Engine

## 1. Introdução

Este documento serve como um guia técnico e de uso para a **BRX Engine**, uma engine de jogos 2D/3D desenvolvida com foco em modularidade e extensibilidade, inspirada na Godot Engine. O principal objetivo é fornecer uma base robusta para o desenvolvimento de jogos com perspectiva 3D completa (6 eixos de liberdade), otimizada para ser utilizada e expandida por inteligências artificiais de programação, como o Codex.

## 2. Visão Geral da Arquitetura

A BRX Engine é estruturada em um **Núcleo (Core)** e **Módulos (Modules)**. O Núcleo fornece as funcionalidades essenciais e a estrutura para a interconexão dos módulos, enquanto os módulos estendem a funcionalidade da engine para áreas específicas como renderização, gerenciamento de cena e física.

   BRX Engine: A simple 3D game engine core written in Python using Pygame and PyOpenGL.

## 3. Estrutura de Classes

O arquivo `brx_engine_core.py` define as seguintes classes principais:

### `Vector3`

Representa um vetor 3D com componentes `x`, `y`, `z`. Inclui operações básicas como adição, subtração, multiplicação por escalar, cálculo de comprimento (`length()`) e normalização (`normalize()`).

**Exemplo de Uso:**

```python
v1 = Vector3(1, 2, 3)
v2 = Vector3(4, 5, 6)
v3 = v1 + v2  # Vector3(5, 7, 9)
```

### `Node`

Classe base para todos os objetos na árvore de cena. Cada `Node` pode ter um `parent` e múltiplos `children`, formando uma hierarquia. Possui propriedades para `position`, `rotation` e `scale` (todas `Vector3`).

**Métodos Principais:**

*   `add_child(child)`: Adiciona um nó filho.
*   `update(delta_time)`: Método virtual para lógica de atualização (chamado recursivamente nos filhos).
*   `render()`: Método virtual para renderização (chamado recursivamente nos filhos), aplicando transformações OpenGL.
*   `_on_render()`: Método a ser sobrescrito por subclasses para desenhar o conteúdo específico do nó.

### `SpatialNode`

Subclasse de `Node` especificamente para objetos que possuem transformações 3D no espaço. Atualmente, herda diretamente de `Node` e serve como um marcador sem adicionar funcionalidades extras, mas é um ponto de extensão para futuras características 3D.

### `Camera`

Subclasse de `SpatialNode` que representa uma câmera de visualização 3D. Define o campo de visão (`fov`), proporção (`aspect`), planos de corte `near` e `far`.

**Métodos Principais:**

*   `apply()`: Configura a matriz de projeção e a matriz de visão do OpenGL com base nas propriedades da câmera.

### `Cube`

Subclasse de `SpatialNode` que representa um cubo 3D simples. Sobrescreve `_on_render()` para desenhar um cubo colorido usando `GL_QUADS`.

### `BRXEngine`

A classe principal da engine, responsável pela inicialização do Pygame e OpenGL, gerenciamento do loop principal, processamento de eventos de input e orquestração da árvore de cena.

**Métodos Principais:**

*   `__init__(width=800, height=600, title="BRX Engine")`: Inicializa a janela, OpenGL e a cena raiz. Cria uma câmera padrão.
*   `run()`: Inicia o loop principal da engine, processando eventos, atualizando a cena e renderizando.

## 3. Guia de Uso para IA (Codex)

O Codex pode interagir com a BRX Engine seguindo a estrutura de classes e métodos definidos. Abaixo estão exemplos e diretrizes para estender a engine.

### 3.1. Criando e Adicionando Objetos à Cena

Todos os objetos visíveis ou lógicos devem ser instâncias de `Node` ou suas subclasses e adicionados à árvore de cena, começando pelo `engine.root`.

**Exemplo:**

```python
# Criar uma nova cena (ou usar a raiz existente)
scene_root = Node("MyScene")
engine.root.add_child(scene_root)

# Criar um novo cubo
my_cube = Cube("MyAwesomeCube")
my_cube.position.x = 3.0
my_cube.rotation.y = 90.0
scene_root.add_child(my_cube)

# Criar outro nó espacial
empty_spatial = SpatialNode("Container")
empty_spatial.position.z = -5.0
scene_root.add_child(empty_spatial)

# Adicionar um cubo como filho do nó espacial
another_cube = Cube("ChildCube")
another_cube.scale = Vector3(0.2, 0.2, 0.2)
empty_spatial.add_child(another_cube)
```

### 3.2. Manipulando Transformações (Posição, Rotação, Escala)

As propriedades `position`, `rotation` e `scale` de qualquer `Node` (ou subclasses) são instâncias de `Vector3` e podem ser modificadas diretamente.

*   **Posição**: `node.position.x`, `node.position.y`, `node.position.z`
*   **Rotação**: `node.rotation.x`, `node.rotation.y`, `node.rotation.z` (em graus, Euler angles).
*   **Escala**: `node.scale.x`, `node.scale.y`, `node.scale.z`

**Exemplo:**

```python
# Mover um cubo
my_cube.position.x += 0.1

# Rotacionar um objeto em torno do eixo Y
my_cube.rotation.y += 1.0

# Redimensionar um objeto
my_cube.scale = Vector3(2.0, 2.0, 2.0)
```

### 3.3. Criando Novas Formas 3D

Para criar novas formas 3D, o Codex deve criar uma nova subclasse de `SpatialNode` e sobrescrever o método `_on_render()` para desenhar a geometria usando as funções OpenGL (`glBegin`, `glVertex3f`, `glColor3f`, `glEnd`, etc.).

**Exemplo (para um Triângulo):**

```python
class Triangle(SpatialNode):
    def _on_render(self):
        glBegin(GL_TRIANGLES)
        glColor3f(1.0, 0.0, 0.0) # Vermelho
        glVertex3f(0.0, 0.5, 0.0)
        glColor3f(0.0, 1.0, 0.0) # Verde
        glVertex3f(-0.5, -0.5, 0.0)
        glColor3f(0.0, 0.0, 1.0) # Azul
        glVertex3f(0.5, -0.5, 0.0)
        glEnd()

# Adicionar um triângulo à cena
my_triangle = Triangle("ColorfulTriangle")
my_triangle.position.z = -2.0
engine.root.add_child(my_triangle)
```

### 3.4. Interação com a Câmera

A câmera principal é acessível via `engine.camera`. Suas propriedades `position` e `rotation` podem ser ajustadas para controlar a visão do mundo.

**Exemplo:**

```python
# Mover a câmera para trás
engine.camera.position.z += 0.5

# Rotacionar a câmera para olhar para baixo
engine.camera.rotation.x = 30.0
```

### 3.5. Processamento de Input

O loop principal da engine já processa eventos Pygame. O Codex pode adicionar lógica de input personalizada dentro do loop `run()` da classe `BRXEngine` ou criar um módulo de input separado.

**Exemplo (dentro do `run()`):**

```python
            for event in pygame.event.get():
                if event.type == pygame.QUIT:
                    self.running = False
                if event.type == pygame.KEYDOWN:
                    if event.key == K_SPACE:
                        print("Espaço pressionado!")
                        # Adicionar lógica de jogo aqui
```

### 3.6. Extensão com Novos Módulos

Para funcionalidades mais complexas (ex: sistema de física avançado, carregamento de modelos 3D), o Codex deve criar novos arquivos Python que implementem essas funcionalidades como módulos, seguindo o padrão de classes e herança da engine. Esses módulos podem ser importados e integrados ao loop principal ou à árvore de cena.
